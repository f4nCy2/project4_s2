# D435i 避障算法原理：点云栅格 + VFH 局部规划

本文档介绍 `src/obstacle_avoidance.py` 使用的几何式局部路径规划方法。该方法将深度图转为相机坐标系下的三维点云，经过倾斜补偿和地面滤除后，投影为二维鸟瞰占用栅格，再使用简化的 **Vector Field Histogram（VFH）** 算法选择行进方向与速度。

---

## 1. 深度图采集

使用 Intel RealSense D435i 捕获 640×480 深度图，帧率 30 fps。在传入后续处理前，依次应用 RealSense 内置滤波：

- **Spatial Filter**：平滑深度图并填充小孔，保留边缘。
- **Temporal Filter**：利用多帧信息抑制时域抖动。
- **Hole Filling Filter**：补全无效深度像素。

---

## 2. 点云生成

通过相机内参将深度图反投影为相机坐标系下的点云 `xyz`，坐标定义如下：

- `x`：水平向右
- `y`：垂直向下
- `z`：光轴向前（深度方向）

每个有效像素 `(u, v, d)` 转换为：

```python
x = (u - cx) * d / fx
y = (v - cy) * d / fy
z = d
```

---

## 3. 倾斜补偿

相机安装时通常具有一定俯角。`CAM_TILT` 表示相机光轴相对水平面向下的倾斜角度（正值为俯视）。通过绕 X 轴旋转点云，使 `y` 轴恢复为竖直向上方向：

```python
xyz_level = compensate_tilt(xyz, tilt_deg=CAM_TILT)
```

这样可保证后续地面滤除使用统一的世界竖直方向。

---

## 4. 地面滤除

已知相机离地高度 `CAM_HEIGHT`，将低于地面一定净空高度的点视为地面点并剔除：

```python
height_thr = -CAM_HEIGHT + GROUND_CLEARANCE   # 默认约 -0.33m
obstacle_points = xyz_level[xyz_level[:, 1] > height_thr]
```

保留高于地面的点作为候选障碍物点。

---

## 5. 占用栅格

将障碍物点投影到水平地面平面，构建固定大小的二维占用栅格：

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `GRID_RES` | 0.05 m | 栅格分辨率 |
| `GRID_SIZE` | 80 | 栅格尺寸（80×80） |
| 覆盖范围 | 4 m × 4 m | 向前 4 m，左右各 ±2 m |

```python
grid = build_occupancy_grid(obstacle_points, cfg)
```

### 机器人 footprint 膨胀

为了将机器人半径纳入规划，使用椭圆形态学核对占用栅格进行膨胀：

```python
kernel = ellipse(w=ROBOT_R, h=ROBOT_R)
dilated_grid = cv2.dilate(grid, kernel)
```

默认 `ROBOT_R = 0.25 m`，即把机器人近似为半径 25 cm 的圆。膨胀后，栅格中标记为障碍的区域已包含机器人轮廓，规划时只需判断栅格是否被占用即可。

---

## 6. VFH 方向选择

将机器人前方 180° 范围划分为 36 个扇区，每个扇区 5°，覆盖方向从 −90°（左）到 +90°（右）。

### 6.1 扇区射线探测

对每个扇区，从机器人所在栅格位置（`row=0, col=grid_center`）沿该方向射线前进，记录能够到达的最大无障碍距离：

```python
free_distance[sector] = ray_cast_max_distance(grid, angle)
```

若 `free_distance < ROBOT_R + SAFE_DIST`，则该扇区被标记为阻塞。

### 6.2 山谷提取与评分

将连续的无障碍扇区组合成“山谷”（valley）。宽度小于 `SECTOR_MIN_WIDTH`（默认 3 个扇区 = 15°）的狭窄山谷被丢弃。

对每个候选山谷按以下公式评分：

```
score = avg_depth * 3.0 + valley_width * 0.2 - |center - TARGET_ANGLE| * 0.5
```

- `avg_depth`：山谷内的平均无障碍深度，越大越优。
- `valley_width`：山谷宽度，越宽越优。
- `|center - TARGET_ANGLE|`：山谷中心偏离目标方向（默认正前方 0°）的代价。

最终选择得分最高的山谷中心方向作为转向角 `steering_angle`。

### 6.3 速度与倒车

根据所选山谷的平均深度，将前进速度映射为三个档位：

| 条件 | 速度 | 含义 |
|------|------|------|
| `avg_depth > 2.0 m` | `V_FAST` = 0.50 m/s | 快速前进 |
| `avg_depth > 1.0 m` | `V_MID` = 0.30 m/s | 中速前进 |
| 其他 | `V_SLOW` = 0.15 m/s | 低速前进 |
| 无可用扇区 | `V_BACK` = −0.15 m/s | 后退 / 等待 |

## 8. 参数调优

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `CAM_HEIGHT` | 0.45 m | 相机离地高度 |
| `CAM_TILT` | +10° | 相机俯角，正值为向下看 |
| `GRID_RES` | 0.05 m | 栅格分辨率 |
| `GRID_SIZE` | 80 | 栅格尺寸 |
| `ROBOT_R` | 0.25 m | 机器人半径，用于栅格膨胀 |
| `SAFE_DIST` | 0.50 m | 安全余量 |
| `NUM_SECTORS` | 36 | VFH 扇区数量 |
| `SECTOR_MIN_WIDTH` | 3 | 最小可通行山谷宽度 |
| `TARGET_ANGLE` | 0° | 期望行进方向 |

### 调优建议

- **室内狭窄环境**：增大 `GRID_RES`、减小 `GRID_SIZE` 以提升实时性；减小 `ROBOT_R` 和 `SAFE_DIST`。
- **室外开阔场景**：增大 `GRID_SIZE` 和 `SAFE_DIST`；`NUM_SECTORS` 保持 36 即可获得 5° 分辨率。
- **地面点过多/过少**：调整 `CAM_TILT` 和 `GROUND_CLEARANCE`。

---

## 9. 局限性与扩展

### 局限性

1. **地面假设**：假设地面为水平平面，对斜坡或楼梯场景需要额外处理。
2. **2.5D 表达**：只能处理凸出的障碍物，无法建模悬空或垂悬结构。
3. **传感器噪声**：透明/反光/纹理缺失表面会导致深度空洞，影响栅格质量。
4. **纯局部规划**：没有全局目标引导，容易陷入局部最优或 U 型区域。

### 扩展方向

- **坡度地面**：将地面建模为平面而非固定高度阈值。
- **三维避障**：保留完整点云或使用体素栅格，支持无人机等三维运动。
- **全局路径融合**：将 VFH 输出接入 ROS `cmd_vel`，结合全局路径规划器行驶。
- **动态障碍物预测**：引入时序信息，预测移动障碍物的未来位置。
